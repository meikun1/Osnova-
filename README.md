# Osnova — Bot Manager + Domain Binding

Менеджер Telegram-ботов на **aiogram 3.x** с встроенной системой привязки
пользовательских доменов через Cloudflare API + Caddy. Один бот-менеджер
управляет неограниченным числом дочерних ботов: обработка заявок в канал,
защита от бана, рассылки, шаблоны, мини-апп по постоянной `startapp`-ссылке.

## Главная фишка

Каждый пользователь привязывает к своему мини-аппу **собственный домен**
прямо из чата с ботом. CF-токены берутся из пула админа — юзер не видит
Cloudflare вообще. Процесс:

1. Юзер шлёт боту домен → бот создаёт зону в CF (из пула аккаунтов),
   ставит A-запись, SSL Strict, прописывает блок в Caddy и возвращает
   2 NS-сервера.
2. Юзер прописывает NS у регистратора домена.
3. Бот в фоне ждёт распространения NS и выпуска сертификата → **сам
   присылает уведомление** «🔒 SSL выпущен».
4. Только после этого разблокируется добавление токена бота.

Подробности процесса для пользователя и для админа — в [`DOMAIN_BIND.md`](DOMAIN_BIND.md).

## Возможности

**Главное меню**
- Список ваших ботов · ⚙️ Создать бота · 🗂 Управление · 📨 Настройки добавления ·
  📣 Рассылка по токенам · 📁 Папки ботов
- Для админов сверху — **🛠 Админ-панель** (управление пулом CF-аккаунтов)

**Создание бота — двухэтапное**
- 🌐 **Привязать домен** — Cloudflare + Caddy + ожидание SSL
- 🤖 **Добавить токен** — токен от @BotFather (валидируется через `getMe`)
- ✅ **Проверить и создать** — добавляет бота, только если оба пункта зелёные

**Карточка бота**
- 🔄 Обновить статистику · 🔌 Перезапуск · 🔥 Защита от бана ·
  📨 Рассылка · 📈 Статистика · 📋 Изменить шаблон · ⚙️ Настройки ·
  🗑 Удалить

**Настройки бота**
- 🔗 Прямая ссылка — мини-апп по постоянной `startapp`-ссылке. URL для
  BotFather (Configure Mini App → Edit Mini App URL → Main App) бот собирает
  автоматически из привязанного домена: `https://<домен>/app/<tg_id>`.
- ✏️ Имя бота — смена отображаемого имени через `setMyName`

**Защита от бана (Guard)**
- Бот отвечает только тем, кто пришёл по персональной deep-link ссылке
  `https://t.me/<bot>?start=<секрет>`; на обычный `/start` молчит.

**Админ-панель**
- Статы пула CF-аккаунтов (всего / свободно / занято)
- ➕ Добавить CF-токены — несколько через запятую/пробелы/строки; дубли
  пропускаются автоматически; токены с не-ASCII символами отклоняются с
  понятной ошибкой; синкается одновременно в БД и в `cf_pool.json`.
- 🧹 Очистить пул — снести всё одной кнопкой (для теста)

## Редактор шаблона мини-апп

«📋 Изменить шаблон» → редактор «Стандартного шаблона мини-апп»
(данные и дефолты в `miniapp_template.py`):

- **Разделы** — тексты ответов бота: /start, кнопка запуска мини-аппа,
  Второе сообщение, Просроченный вход, Успешная авторизация, Автоспам и др.
- **Страницы мини-аппа** — 4 листа (Главная / Ввод кода / 2FA / Успех)
  со своими под-полями (эмодзи, текст кнопки, плейсхолдер, подсказка, текст).
- **Действия / оформление** — 🎭 Вид (5 вариантов вёрстки), 🖼 Фон,
  💨 Блюр, 🎨 Цвет интерфейса.

## Структура

```
bot.py                  — точка входа (менеджер + рантайм + веб-часть)
config.py               — конфиг из env (включая пул CF, домены, админов)
database.py             — синхронный слой Postgres/SQLite
keyboards.py            — главное меню (с кнопкой Админ-панели)
domain_flow.py          — Cloudflare API + Caddy (привязка/удаление доменов)
cf_pool_loader.py       — импорт пула из cf_pool.json + обратный синк
ssl_watcher.py          — фоновый воркер: проверяет SSL и шлёт уведомление
templates.py            — типы шаблонов (standard/text/welcome)
miniapp_template.py     — данные «Стандартного шаблона мини-аппа»
directlink_service.py   — связка «Прямая ссылка» с БД
handlers/               — роутеры меню и модулей
handlers/admin.py       — админ-панель (пул CF)
handlers/domain.py      — привязка домена (FSM)
handlers/create_bot.py  — двухэтапное создание бота (домен + токен)
child/                  — рантайм дочерних ботов
direct_link/            — модуль «Прямая ссылка» (FastAPI + middleware)
web/app.py              — FastAPI мини-аппа + /static (стикер)
web/static/             — анимация ввода кода (gif)
web/model/              — 3D-модель (для preview3d)
Caddyfile               — минимальный стартовый конфиг
Dockerfile.caddy        — кастомный билд Caddy с github.com/caddy-dns/cloudflare
docker-compose.yml      — полный стек одной командой
cf_pool.example.json    — шаблон пула CF-токенов
DOMAIN_BIND.md          — инструкция привязки доменов (юзер + админ)
SERVER_SETUP.md         — пошаговый деплой на сервер
```

## Деплой одной командой (рекомендуется)

Полная инструкция в [`SERVER_SETUP.md`](SERVER_SETUP.md). Кратко:

```bash
git clone https://github.com/meikun1/Osnova-.git
cd Osnova-

cp .env.docker.example .env
nano .env                        # 5 обязательных строк

cp cf_pool.example.json cf_pool.json
nano cf_pool.json                # вставить CF API Token(ы)

docker compose up -d --build
```

Поднимаются три контейнера:
- `app` — бот-менеджер + веб-часть мини-аппа (полинг + FastAPI).
- `caddy` — кастомный билд с DNS-плагином Cloudflare; admin API на `:2019`
  доступен только во внутренней сети compose. Бот делает горячий reload
  через HTTP POST `/load` без перезапуска контейнера.
- `db` — Postgres 16 (хранит ботов, пул CF, привязки доменов, шаблоны).

## Запуск локально (без Docker)

```bash
pip install -r requirements.txt
cp .env.example .env             # MANAGER_BOT_TOKEN и пр.
export $(grep -v '^#' .env | xargs)
python bot.py
```

Дочерние боты поднимаются автоматически при старте и при добавлении.
Без Caddy/CF-пула привязка доменов работать не будет — это режим только для
разработки самого менеджера.

## Переменные окружения

### Обязательные

| Переменная | Что |
|---|---|
| `MANAGER_BOT_TOKEN` | Токен бота-менеджера от @BotFather |
| `DOMAIN_SERVER_IP` | Публичный IPv4 сервера (на него идут A-записи) |
| `CADDY_ACME_EMAIL` | Email для регистрации в Let's Encrypt |
| `POSTGRES_PASSWORD` | Пароль для встроенного Postgres (compose) |
| `DIRECT_LINK_SESSION_SECRET` | `openssl rand -hex 32` |
| `ADMIN_IDS` | Telegram-id админов через запятую |

### Опциональные

| Переменная | Дефолт | Что |
|---|---|---|
| `CF_POOL_JSON_PATH` | `cf_pool.json` | Путь к пулу CF-токенов |
| `DOMAIN_CADDYFILE` | `/etc/caddy/Caddyfile` | Caddyfile для прописывания доменов |
| `DOMAIN_CADDY_ADMIN_URL` | `http://caddy:2019` | Admin API caddy для reload |
| `DOMAIN_TARGET` | `app:8080` | Куда reverse_proxy шлёт трафик |
| `DOMAIN_BIND_GUIDE_URL` | GitHub URL | Куда ссылается «📖 Инструкция» |
| `CODE_FAIL_REDIRECT_URL` | — | Куда увести юзера после 3 неверных кодов |
| `MINIAPP_BASE_URL` | — | Бэкап-домен (legacy; обычно не нужен) |
| `RUN_WEB` | `0` (compose: `1`) | Поднимать ли FastAPI мини-аппа |
| `BAN_CHECK_INTERVAL` | `15` | Период проверки бана (сек) |

## Заметки

- **БД** — встроенный Postgres в docker-compose (контейнер `db`).
  `database.py` — единый синхронный API. Все таблицы и миграции создаются
  автоматически при `init_db()`. Данные в volume `db_data` — переживают
  `docker compose down` без `-v`.
- **Гео** в статистике берётся из `language_code` пользователя.
- **Стикер** на странице ввода кода — `web/static/code-sticker.gif`,
  отдаётся через `app.mount("/static", ...)`.
- **TLS-сертификаты Caddy** хранятся в named volume `caddy_data` —
  переживают `docker compose down` без `-v`.
- **`cf_pool.json`** в `.gitignore`; пример формата — `cf_pool.example.json`.
- На запросы CF API бот идемпотентен: если зона/A-запись уже есть, повторная
  привязка проходит без ошибок (для дешёвого retry).
