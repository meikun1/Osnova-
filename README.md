# Bot Manager

Менеджер Telegram-ботов на **aiogram 3.x**. Один бот-менеджер управляет
несколькими дочерними ботами (добавленными по токену из @BotFather):
обработка заявок в канал, защита от бана, статистика, рассылки, шаблоны
и модуль «Прямая ссылка» (мини-апп по постоянной `startapp`-ссылке).

## Возможности

**Главная страница**
- Список ваших ботов
- Обработка заявок в закрытый канал (бот пишет человеку первым)
- Управление ботами · Создать бота · Настройки добавления · Рассылка по токенам · Папки ботов

**Страница бота** (карточка)
- Шапка: имя, токен, шаблон, статус защиты, ссылка для юзера, статус мини-аппа
- Модули: 🔄 Обновить статистику · 🔌 Перезапуск · 🔥 Защита от бана ·
  📨 Рассылка · 📈 Статистика · 📋 Изменить шаблон · ⚙️ Настройки ·
  🗑 Удалить · ⬅️ Назад

**Настройки бота**
- 🔗 Прямая ссылка — мини-апп по постоянной `startapp`-ссылке
  (генерируется один раз на бота). При включении бот перестаёт реагировать
  на `/start`; вход без правильной ссылки редиректит на сторонний ресурс.
- ✏️ Имя бота — смена отображаемого имени (через `setMyName`)

**Защита от бана (Guard)**
- Бот отвечает только тем, кто пришёл по персональной deep-link ссылке
  `https://t.me/<bot>?start=<секрет>`; на обычный `/start` молчит.

## Редактор шаблона мини-апп

«📋 Изменить шаблон» → выбранный шаблон открывает редактор «Стандартного
шаблона мини-апп». Он объединяет три части (перенесены из автономного
бота-конструктора, данные и дефолты — в `miniapp_template.py`):

- **Разделы** — тексты и подписи кнопок ответов бота: Ответ на /start,
  Кнопка запуска мини-апп, Второе сообщение, Просроченный вход и кнопка,
  Успешная авторизация, Автоспам (авторизованным/неавторизованным),
  Пост админ канала, Показ авторизации, Показ кода.
- **Листы мини-аппа** («📱 Страницы мини-апп») — 4 страницы, у каждой свои
  под-поля:
  - Главная: эмодзи · текст кнопки · текст ожидания · текст;
  - Ввод кода: эмодзи · текст кнопки · текст неверного кода · текст;
  - 2FA: эмодзи · текст кнопки · текст в поле ввода · подсказка · текст;
  - Успех: эмодзи · текст кнопки · текст.
- **Действия / оформление** — 🎭 Вид (5 вариантов вёрстки страниц:
  Классический / Карточка / Стекло / Минимал / Кнопка снизу), 🖼 Фон
  (свой фон по ссылке), 💨 Блюр, 🎨 Цвет интерфейса (палитра из 10 вариантов).

Каждое поле редактируется отдельно (✏️ Изменить / 🗑 Очистить). Незаданные
поля показывают дефолт из `miniapp_template.ALL_DEFAULTS`; очистка поля
оставляет его пустым (бот/мини-апп используют свой запасной текст). Тексты
разделов и страниц хранятся в JSON-`content` шаблона: разделы — по своему
ключу, под-поля страниц — по ключу `{page}_{field}` (напр. `main_text`,
`code_wrong`, `twofa_placeholder`). Эти значения читают рантайм дочернего
бота (`child/runner.py`) и страница мини-аппа (`web/app.py`).

## Структура

```
bot.py                  — точка входа (менеджер + рантайм + опц. веб)
config.py               — конфиг из переменных окружения
database.py             — SQLite-слой (синхронный)
keyboards.py            — главное меню
templates.py            — типы шаблонов дочерних ботов (standard/text/welcome)
miniapp_template.py     — данные и дефолты «Стандартного шаблона мини-аппа»
                          (разделы, листы-страницы, под-поля, палитра цветов)
directlink_service.py   — связка модуля «Прямая ссылка» с SQLite
handlers/               — роутеры менеджера (страницы и модули)
handlers/template.py    — редактор «Стандартного шаблона мини-апп»
child/                  — рантайм и логика дочерних ботов
direct_link/            — модуль «Прямая ссылка» (FastAPI + aiogram middleware)
web/app.py              — FastAPI мини-аппа (опционально)
```

## Запуск локально

```bash
pip install -r requirements.txt
cp .env.example .env        # впишите MANAGER_BOT_TOKEN
export $(grep -v '^#' .env | xargs)   # или используйте python-dotenv / direnv
python bot.py
```

Дочерние боты поднимаются автоматически при старте и при добавлении.
Кнопка «Перезапуск» перезапускает polling выбранного бота.

## Деплой на Railway (бот-воркер + Neon)

Менеджер работает в режиме **polling**, поэтому это обычный worker-процесс
(HTTP-порт не нужен, пока `RUN_WEB=0`).

1. **База — Neon.** Создайте проект на [neon.tech](https://neon.tech),
   скопируйте connection string (psql) — он вида
   `postgresql://user:pass@ep-xxx.neon.tech/dbname?sslmode=require`.
2. **Создайте сервис на Railway** из этого репозитория (Deploy from GitHub
   repo). Railway соберёт проект через Nixpacks, старт-команда уже задана в
   `railway.json` → `python bot.py`.
3. **Переменные окружения** (Railway → Variables):
   - `MANAGER_BOT_TOKEN` — токен бота-менеджера из @BotFather
   - `DATABASE_URL` — connection string из Neon
   - `DIRECT_LINK_SESSION_SECRET` — длинная случайная строка
   - (опц.) `DIRECT_LINK_REDIRECT_URL`, `DIRECT_LINK_MANUAL_URL`
4. **Деплой.** В логах должно появиться «Менеджер запущен».

> Таблицы создаются автоматически при первом старте (`init_db`).
> Так как БД — Neon, данные переживают редеплои и рестарты.

Файлы для Railway: `railway.json` (старт-команда), `Procfile` (worker),
`.python-version` (3.11), `requirements.txt`.

### Мини-апп «Прямая ссылка»

Для боевой работы мини-аппа нужен HTTPS-домен, указанный в @BotFather
(Web App URL). Поднимите веб-часть: `RUN_WEB=1 python bot.py`.
Эндпоинты публикуются под `/dl/...` (см. `direct_link/README.md`).

## Заметки

- БД — PostgreSQL/Neon (если задан `DATABASE_URL`) или SQLite (иначе).
  `database.py` — единый синхронный API над обоими бэкендами.
- «Гео» в статистике берётся из `language_code` пользователя — лучшее
  доступное приближение без запроса геолокации.
